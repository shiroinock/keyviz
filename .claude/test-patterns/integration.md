# テストパターン: 統合テスト

## 対象
- `src/__tests__/integration/` に配置
- 複数モジュールの連携、エンドツーエンドのシナリオ

## 特徴
- **テスト容易性**: 低〜中（モック最小限）
- **TDD方式**: テストレイター
- **配置**: separated（`src/__tests__/integration/` に分離）

## テストの構造

```typescript
import { describe, test, expect } from 'vitest';

describe('統合テスト: シナリオ名', () => {
  test('一連のフローが動作する', () => {
    // 複数モジュールの連携をテスト
  });
});
```

## カバーすべきシナリオ
1. **ユーザーシナリオ**: 実際の使用フロー全体
2. **モジュール連携**: コンポーネント ↔ ユーティリティ
3. **データフロー**: 入力 → 変換 → 表示

## モックの最小化
統合テストではモックは最小限に。外部依存のみモック。
