# コード規約スキル

## 基本原則

### 1. マジックナンバーの排除
ドメイン知識としてみなせる数値は、必ず定数として定義する。

### 2. 定数参照の原則
テストファイルでも定数をインポートして使用する。

### 3. パフォーマンス最適化
繰り返し処理で共通の操作は、ループ外に抽出する。

### 4. 型安全性の原則
型アサーション（`as`）は可能な限り避け、型ガードを使用する。

### 5. マジック文字列の定数化
繰り返し使用される文字列リテラルは定数として定義する。

### 6. 重複コードの排除
同じロジックが複数箇所に存在する場合は、関数として抽出する。

### 7. ヘルパー関数の配置と命名
- 型ガード関数: `is〜`（例: `isValidConfig`）
- 取得関数: `get〜`（例: `getDefaultConfig`）
- 変換関数: `to〜`（例: `toJSON`）

## チェックリスト

- [ ] ドメイン知識としてみなせる数値が定数として定義されているか

- [ ] 型アサーション（`as`）を避け、型ガードを使用しているか
- [ ] マジック文字列を定数化しているか
- [ ] 重複コードを関数として抽出しているか
